53 /* Creates a ready to use arena, and configures the arena to support a chunk_sz.
54  *
55  * We consider how malloc operates carefully to optimize block sizes and thereby improves
56  * efficiency of the heap.
57  * * The amount of memory that malloc needs to store a pointer in the block is subtracted
58  * from the requested memory.
59  * * We do this, so that it is easy to use block sizes the size of page size (4096), whole
60  * multiples of that, or whole parts of that, (2048, 1024, 512, 256, 128, 64.). This makes
61  * it faster to allocate blocks, and if malloc needs to split blocks delivered from the
62  * heap, then the heap is kept as tidy as posible.
63  * * I recommend the smallest chunk_sz requested to be 1024 bytes.
64  */
