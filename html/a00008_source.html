45 /* Creates a ready to use arena, and configures the arena to support a chunk_sz.
46  *
47  * We consider how malloc operates carefully to optimize block sizes and thereby improves
48  * efficiency of the heap.
49  * * The amount of memory that malloc needs to store a pointer in the block is subtracted
50  * from the requested memory.
51  * * We do this, so that it is easy to use block sizes the size of page size (4096), whole
52  * multiples of that, or whole parts of that, (2048, 1024, 512, 256, 128, 64.). This makes
53  * it faster to allocate blocks, and if malloc needs to split blocks delivered from the
54  * heap, then the heap is kept as tidy as posible.
55  * * I recommend the smallest chunk_sz requested to be 1024 bytes.
56  */
