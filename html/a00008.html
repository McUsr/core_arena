Detailed Description

A general purpose arena allocation strategy that is based on malloc.

It is simple, you can't resize previously allocated memory, and it shouldn't break during production, if it breaks, then we exit with an EXIT_FAILURE. So, if you need a hashtable, then you should allocate the resizeable array with calloc, so you can resize that with realloc, and allocate the hashes from the arena. This arena allocation strategy uses malloc intrinisically, if you want other schemes than malloc provides you should look elsewhere.
